## 问题 1：嫌疑人角色错位（已修复 ✅）

### 现象
嫌疑人 R2 回答中出现了侦探应说的审讯台词：
```
"侦探问[那你能否告诉我，在案发时间段内，也就是晚上10点到12点之间，
你具体在什么地方？是否有目击者可以证明你的行踪？]
现在轮到我作为嫌疑人李建国回答这个问题。
我在家睡觉，没有离开过那里，也没有人能证明我的行踪。"
```

### 根因
LLM 生成嫌疑人回答时，先复述了侦探的提问（扮演侦探），再以嫌疑人身份作答。本质是角色上下文隔离不足，模型没有清晰的"你是谁、你只说什么"约束。

### 影响
- **严重性**: 🔴 Critical
- **影响范围**: 核心玩法崩溃，嫌疑人面板输出侦探台词，游戏不可玩

### 修复方案
**文件**: `src/interrogation_mvp/llm_gateway.py`

1. **System Prompt 强化（层1）**  
   - 嫌疑人规则新增：明确区分"允许的防御性提问"和"禁止的审讯式提问"  
   - 新增正确/错误示例，直接对应问题场景  
   - 之前方案"禁止所有问句"过于激进（嫌疑人说"我能请律师吗？"是合理的）  
   - 最终规则：禁止替侦探提审讯问（如"你具体在哪里""你在什么时候出去的"），允许防御性提问（如"我能看一下证据吗"）

2. **后置清洗强化（层3）**  
   - `_clean_speech(text, role)` 新增针对嫌疑人的审讯式提问正则清洗  
   - 触发全截取的阈值从 2 个问号提高到 3 个（给防御性提问预留空间）  
   - 白名单：含"我能""我可以""您"的问句被视为合理防御保留

### 验证结果
- ✅ 8/8 llm_gateway 测试通过
- ✅ 修复后重新启动 GUI，待观察

## 问题 2：矛盾检测误判（待修复 ⏳）

### 现象
底部矛盾区域在 R2 时显示：
```
⚠️ [TIME/MEDIUM] 时间线自述矛盾：既称外出又称固定时段在家。
```
但此时嫌疑人 **从未承认外出**，R1 说"不知道王某什么时候到河边"，R2 说"在家睡觉"，两句话并不矛盾。

### 根因
`contradiction.py` 的时间线矛盾规则触发条件过宽，在"不在场声明"出现时就激活，而没有要求先存在"明确的外出承认"。规则匹配到了原文中包含"外出"语义的词汇，但上下文语义并不支持矛盾成立。

### 影响
- **严重性**: 🟡 Medium
- **影响范围**: 用户体验受损，虚假矛盾降低可信度，玩家无法准确判断审讯进展

### 待修复方案
**文件**: `src/interrogation_mvp/contradiction.py`

建议调整方向：
1. 时间线矛盾需要至少两条历史陈述相互冲突才触发，不能对单条陈述触发
2. 提高触发阈值：同一类矛盾需要在不同回合分别出现相反说法
3. 增加置信度过滤：低置信度矛盾不显示，或标注"疑似"

**优先级**: P1（不阻塞核心玩法，但影响信任感）  
**建议**: 在 Week 4 内部测试期间修复，进入用户测试前必须解决

## 修复过程衍生发现

### 发现 1：角色限制设计陷阱
初始修复方案使用"禁止嫌疑人使用问句"，但这会误伤合理的防御性提问（如"我能请律师吗？""您说的证据是什么？"）。  
**结论**：角色限制规则需要精确到"禁止替对方角色提问"，而不是"禁止自己提问"。

### 发现 2：后置清洗的局限性
正则清洗可以处理已知模式的角色错位，但对"模型在回答前先复述侦探问题"这类结构污染，靠后置清洗代价高且不稳定。根本解法是 System Prompt + User Prompt 层面的角色身份锁定（层1+层2），清洗只作兜底。
